Постановка задачи:
реализовать акцию "Каждый N товар за X рублей". У акции должны быть настройки, где выставляется период активности акции, параметры N и X.

Пути решения:

1. Способ: пишем в result_modifler корзины обработку массива asResult таким образом, чтобы скрипт пересчитывал цены.
	Последовательность работы такого скрипта: 
		1. Проходим циклом по массиву arResult;
		2. Сортируем товары по возрастанию цены (Можно не перезаписывать массив, а объявить, к примеру, новый - чтобы не менять штатную сортировку);
		3. Определяем количество товаров в корзине, делим на N без остатка;
		4. Меняем цену первых N товаров на X.
		
2. Способ: как вариант - обернуть алгоритм способа 2 в компонент, чтобы можно было визуально менять параметры, и подключить его в корзине.
	
3. Способ: зарегистрировать обработчик в init.php, и привязать его к событию добавления товара в корзину (алгоритм из п.1)

4. Способ: штатными средствами Bitrix, а именно - используя Правила работы с корзиной. 
	Порядок действий: 
		1. Переходим по пути ( Маркетинг - Товарный маркетинг - Правила работы с корзиной );
		2. Нажимаем кнопку "Добавить правило";
		3. Задаем имя и при необходимости период активности и правила;
		4. Устанавливаем применимость;
		5. Переходим на вкладку "Действия и условия";
		6. Добавить действие->изменить стоимость товаров в корзине;
		7. Производим следующую настройку действия: Применить скидку X % (или рублей) к товарам, для которых все условия выполнены;
		8. Проивзодим следующую настройку условия: Количество товара в корзине равно M. 
	Примечание - при таком подходе не получится в полной мере реализовать необходимые условия по следующим причинам:
		- скидка устанавливается только на количество товаров в каждой позиции;
		- скидка будет ограничена количеством условий, которые мы зададим. 
		
	p.s.: при поиске информации о решении такой задачи в интернете - ответами было то, что  
	такую скидку штатным функционалом не реализовать
